## 🔐 Безопасность

### BUG-001 🟢 Загрузка .env не автоматическая
**Исправлено:** `dotenv` загружается в `loadConfig()` автоматически. Работает в том числе при cron.

### BUG-002 🟢 Нет валидации ключей при старте
**Исправлено:** `loadConfig()` пишет `warn` если magistrate enabled без ключей (подтверждено smoke-тестом 2026-07-01).

### BUG-003 🟢 config.json доступен по сети через viewer
**Исправлено:** `GET /api/config` возвращает `SafeAppConfig` — без `apiKey`/`fallbackApiKey`, только флаги `primaryKeySet`/`fallbackKeySet`.

### BUG-004 🟢 Отсутствует timeout на парсинг
**Исправлено:** `Promise.race([parse(), timeout(10s)])` в orchestrator.

### BUG-005 🟢 run-log не аккумулирует историю
**Исправлено:** Лог записывается в `run-log-YYYY-MM-DD.json`, добавляется к существующим записям дня.

### BUG-006 🔴 Повторный парсинг стирает данные
**Описание:** Каждый запуск перезаписывает `cases-{court}-{date}.json`. При двух запусках в один день — второй стирает результат первого.
**Решение:** Добавить timestamp в имя файла (`cases-{court}-{date}-{hhmm}.json`) или мержить по uid в существующий файл.
**Файл:** `packages/exporter/json.ts`

### BUG-007 🟢 Нет lock-файла от параллельного запуска
**Исправлено:** `logs/orchestrator.lock` создаётся при старте, удаляется в `finally`.

### BUG-008 🟢 CSS-селекторы не проверены
**Исправлено:** smoke-тест 2026-07-01 подтвердил работу селекторов `district`. `uid=59RS0007-01-2025-005668-17`, судья, стороны, события парсятся корректно.
**Проверить:** appeal и cassation после реализации адаптеров.

### BUG-009 🟢 UID fallback отсутствовал
**Исправлено:** Fallback на `case_uid` из URL, затем `case_id`. Все адаптеры.

### BUG-010 🔴 Нет различения капча/503
**Описание:** Если вернулась страница капчи (HTTP 200 + форма), ошибка в логах будет вводящей в заблуждение.
**Решение:** В `parse()` проверять наличие `<form` с полем капчи, бросать `CaptchaRequiredError`.
**Файл:** все адаптеры.

### BUG-011 🟢 node-fetch ESM + Windows
**Исправлено:** `node-fetch` удалён, встроенный `fetch` Node 22 везде.

### BUG-012 🟢 charset автоопределение
**Исправлено:** charset читается из `Content-Type` заголовка. smoke-тест подтвердил win1251 на всех судах.
